# Frontend Backend Integration PRD

## Project Overview
Update the existing Vue.js frontend application to work seamlessly with the new FastAPI backend architecture. The frontend is already well-structured with Vue 3, TypeScript, PrimeVue, and Pinia, but needs updates to integrate with the new backend API endpoints, authentication system, and advanced features.

## Current Frontend State
- Vue 3 with Composition API and TypeScript
- PrimeVue UI component library
- Pinia state management
- Firebase authentication
- WebSocket support for real-time updates
- Well-organized structure with composables, services, and stores

## Backend Capabilities (New)
- FastAPI with async/await patterns
- Firebase-based user authentication and management
- Document management with URL/content processing
- AI-powered content analysis and entity extraction
- Real-time WebSocket communication
- Advanced search with vector similarity
- Background task processing
- Comprehensive error handling and logging

## Integration Requirements

### 1. API Endpoint Updates
- Update all API calls to match new backend endpoints
- Replace old `/documents_plus/user/{user_id}` with new `/documents/` endpoints
- Update bookmark endpoints to use new `/bookmarks/` structure
- Implement new user management endpoints (`/users/`)
- Add support for new document analysis endpoints

### 2. Authentication Integration
- Integrate Firebase auth with backend user management system
- Update user context handling for API calls
- Implement proper authentication headers and tokens
- Handle user profile management through backend

### 3. Data Models and TypeScript Updates
- Update existing TypeScript models to match new backend response structures
- Create new models for advanced features (analysis reports, entity extraction, etc.)
- Update DocModel, Bookmark, and other core models
- Add models for new backend features

### 4. WebSocket Implementation Enhancement
- Update WebSocket setup to match new backend structure
- Implement support for new message types (analysis progress, entity extraction, etc.)
- Handle real-time document processing updates
- Add support for background task progress tracking

### 5. Service Layer Updates
- Update ProjectService.js to use new backend endpoints
- Enhance DocxrayService.js with new backend capabilities
- Implement proper error handling matching backend responses
- Add new services for advanced features

### 6. UI Components and Features
- Add new UI components for content analysis visualization
- Implement entity extraction display components
- Add document processing status indicators
- Create analysis report viewing components
- Enhance existing document management interface

### 7. State Management Updates
- Update Pinia stores to work with new backend data structures
- Implement proper state management for new features
- Update document store to handle new backend responses
- Add stores for analysis and entity data

### 8. Real-time Features
- Implement real-time document processing updates
- Add progress indicators for background tasks
- Handle WebSocket message routing for different feature types
- Implement proper error handling for real-time updates
