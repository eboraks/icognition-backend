# iCognition Backend - Product Requirements Document

## Executive Summary

iCognition is an AI-powered document analysis and knowledge management platform that enables users to bookmark web pages and PDF documents, automatically extract insights, and engage in intelligent conversations about the content. The platform leverages Google's Gemini AI models for content analysis, entity extraction, and question-answering capabilities.

## Product Overview

### Vision
To create an intelligent web page, articles, and document analysis platform that transforms static content into interactive, searchable knowledge through AI-powered insights and conversational interfaces.

### Mission
Empower users to efficiently process, understand, and interact with digital content by providing automated analysis, entity extraction, and intelligent question-answering capabilities.

## Core Features

### 1. Document Ingestion and Processing

#### 1.1 Web Page Bookmarking
- **Functionality**: Users can bookmark web pages by providing URLs or the content of the web page
- **Processing**:
  - The client should send the page content either in clean text or raw HTML. But sometimes, the client won’t be able to get the page text, in those cases, the client will send URL and the backend will try and fetch the content from the URL.
  - HTML content extraction using BeautifulSoup
  - Meta tag parsing (title, author, publication date, description, keywords)
  - Main article content identification
  - Content validation (minimum paragraph requirements)
- **Storage**: Raw HTML and structured content stored in PostgreSQL
- **Supported Sources**: Web pages, excluding home pages and search pages

#### 1.2 PDF Document Upload - Remove this functionality. 
- **Functionality**: Users can upload PDF files for analysis
- **Processing**:
  - PDF to Markdown conversion using pymupdf4llm
  - Markdown to HTML conversion
  - Content structure preservation
- **Storage**: Files stored in Google Cloud Storage via FUSE mount
- **File Management**: Organized by user ID and source ID

#### 1.3 Content Analysis Pipeline – This pipeline should be broken down into multiple background tasks, rather than having a single large pipeline. 
- **Automated Processing**: Background tasks for content analysis
- **AI-Powered Extraction**:
  - Document summarization - can be removed. 
  - Key bullet points generation - keep 
  - Entity and concept identification - keep, but put in a different background task
  - Topic categorization - keep, but put in a different background task
- **Status Tracking**: Processing states (Pending, Processing, Done, Failure)

### 2. AI-Powered Content Analysis

#### 2.1 Document Summarization
- **Technology**: Google Gemini 2.0 Flash/Pro models
- **Outputs**:
  - Document overview ("What this article is about")
  - Key bullet points (up to 6 points)
  - Content categorization
  - Bias assessment
- **Citations**: Source sentences with relevance scores

#### 2.2 Entity Extraction and Enrichment
- **Entity Types**: People, Products, Companies, Locations, Events, Technology, Topics
- **Enrichment Process**:
  - Wikidata integration for entity validation
  - AI-powered entity matching
  - Description enhancement
  - Alias identification
- **Storage**: Vector embeddings for semantic search

#### 2.3 Question-Answer Generation  – remove this capability
- **Automated Q&A**: AI generates relevant questions and answers
- **Custom Questions**: Users can ask specific questions about documents
- **Citation Support**: Answers include source citations
- **Vector Search**: Question embeddings for similarity matching

### 3. Intelligent Search and Discovery

#### 3.1 Hybrid Search System – Remove this capability
- **Text Search**: PostgreSQL full-text search with TSVECTOR
- **Semantic Search**: Vector similarity using pgvector
- **Search Aggregation**: Combines text and semantic results
- **Threshold Management**: Dynamic similarity thresholds

#### 3.2 Entity-Based Filtering
- **Tree Structure**: Hierarchical entity organization
- **Document Filtering**: Filter documents by entities
- **User-Specific**: Personalized entity trees per user

#### 3.3 RAG (Retrieval-Augmented Generation)
- **Context Retrieval**: Relevant document chunks for questions
- **Answer Generation**: AI-powered responses with citations
- **Multi-Document**: Cross-document question answering
- Reimplement with LangChain and LangGraph

### 4. Conversational Interface

#### 4.1 Chat System
- **Document Chat**: Contextual conversations about specific documents
- **Collection Chat**: Multi-document conversations
- **Session Management**: Persistent chat sessions with timeout
- **Real-time Updates**: WebSocket-based live updates

#### 4.2 Chat Features
- **Suggested Questions**: AI-generated conversation starters
- **Opening Messages**: Automated document introductions
- **Bias Analysis**: Content bias assessment and communication
- **HTML Formatting**: Rich text responses with formatting

#### 4.3 Chat History
- **Persistent Storage**: All conversations saved to database
- **Context Awareness**: Chat history influences responses
- **Event Tracking**: Different message types (summary, entities, Q&A)

### 5. User Management and Data Organization

#### 5.1 User System
- **User Creation**: Automatic user creation on first bookmark
- **User Data**: Personalized document collections
- **Entity Associations**: User-specific entity relationships

#### 5.2 Collections Management – Remove functionality
- **Dynamic Collections**: Collections created from document IDs
- **Hash-based IDs**: Consistent collection identification
- **Multi-Document Support**: Collections can contain multiple documents

#### 5.3 Data Privacy
- **User Isolation**: All data scoped to user ID
- **Secure Storage**: Encrypted data storage
- **Access Control**: User-specific data access

### 6. Technical Infrastructure

#### 6.1 Database Architecture
- **Primary Database**: PostgreSQL with pgvector extension
- **Models**: SQLModel-based ORM
- **Relationships**: Many-to-many entity-document relationships
- **Indexing**: GIN indexes for full-text search, HNSW for vector search

#### 6.2 AI Integration
- **Primary AI**: Google Gemini 2.0 Flash/Pro
- **Embeddings**: Gemini embedding models (768 and 3072 dimensions)
- **Rate Limiting**: Retry logic with exponential backoff
- **Error Handling**: Comprehensive error management

#### 6.3 External Services
- **Wikidata API**: Entity enrichment and validation
- **Google Cloud Storage**: File storage via FUSE mount
- **WebSocket**: Real-time communication
- **Background Tasks**: Async processing with FastAPI

#### 6.4 API Architecture
- **Framework**: FastAPI with async support
- **Endpoints**: RESTful API with comprehensive error handling
- **WebSocket**: Real-time updates and chat
- **CORS**: Cross-origin resource sharing configuration

## User Workflows

### Primary User Journey

1. **Content Discovery**: User finds interesting web page
2. **Bookmark/Upload**: User bookmarks URL 
3. **Automatic Processing**: System processes content in background
4. **AI Analysis**: Document analyzed for entities, topics, summaries
5. **Interactive Exploration**: User engages with suggested questions
6. **Custom Queries**: User asks specific questions about content
7. **Search and Discovery**: User searches across all their documents
8. **Collection Building**: User creates collections of related documents

### Secondary Workflows

- **Entity Exploration**: Browse documents by entities and topics
- **Cross-Document Analysis**: Ask questions across multiple documents
- **Content Regeneration**: Reprocess documents with updated AI models
- **Data Management**: Organize and manage document collections
